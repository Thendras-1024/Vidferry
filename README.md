# Vidferry

Vidferry 是一个面向视频二次创作工作流的本地优先自动化项目，目标是把“找视频、下载视频、处理视频、管理素材、发布到平台”串成一条可持续完善的全链路流水线。

当前项目仍处于开发阶段，重点先打通 YouTube 视频线索获取、`yt-dlp` 下载、AI 字幕处理、FFmpeg 视频生成、素材入库、账号状态管理，以及抖音 / B 站发布工作流。后续会逐步扩展到自动剪辑、标题封面生成、关键节点总结字幕、开头钩子混剪、信息条补充、BGM/音效、OSS 存储等能力。

## 项目目标

Vidferry 不是单纯的视频搬运工具，而是一个逐步演进的自动化视频再创作系统：

- 从 YouTube 等来源发现可用视频线索。
- 下载原始视频并和本地素材库建立映射。
- 基于语音识别、翻译、字幕生成和 FFmpeg 完成视频处理。
- 对账号状态、Cookie、发布平台进行统一管理。
- 把处理后的视频一键发布到抖音、Bilibili 等平台。
- 后续加入自动剪辑能力，让视频具备更明显的二次创作痕迹。

## 当前能力

### 视频线索与下载

- 支持通过关键词查询 YouTube 视频线索。
- 保存视频标题、频道、粉丝数、发布时间、时长、缩略图、链接等信息。
- 支持基于 `yt-dlp` 创建下载任务。
- 支持下载进度、下载速度、任务状态展示。
- 下载目录通过 `.env` 配置，适合本地开发部署。

### AI 字幕与视频处理

- 使用 `faster-whisper` 对视频音频进行语音识别。
- 使用翻译模块生成中文字幕。
- 生成 ASS 字幕文件，并通过 FFmpeg 烧录到视频中。
- 处理完成后自动写入素材库，供发布中心继续使用。

### 素材与任务管理

- 支持下载素材、处理后素材的入库记录。
- 支持 YouTube 视频下载状态、翻译状态、发布状态管理。
- 支持工作流任务列表、任务进度、失败信息查看。

### 账号与发布

- 支持抖音、Bilibili、小红书、快手、视频号等平台的上传能力基础。
- 当前重点接入抖音、Bilibili 发布链路。
- 账号异常时不自动重连，由用户手动重新连接。
- 右上角消息中心会提示账号异常，并支持“已知晓 / 已处理”。
- 重新连接异常账号时会覆盖原账号记录，不再新增重复账号。

## 自动剪辑规划

自动剪辑会按阶段逐步加入，优先实现成本低、效果明显、能稳定落地的能力。

### 第一阶段：二创包装

- 动态标题帧、封面帧、顶部文字框。
- 关键节点中文总结字幕条。
- 开头 3 段钩子混剪。
- 基础调色、锐化、画面比例适配。

### 第二阶段：节奏重构

- 基于字幕时间轴删除冗余停顿。
- 对说话片段加速，对高光反应片段轻微慢放。
- 加入淡入淡出、短黑场等转场。
- 支持 BGM 和简单音效点缀。

### 第三阶段：内容增量

- 根据视频内容生成信息补充条。
- 对文化差异、地点、事件进行中文解释。
- 支持旁白文案生成和 TTS 合成。
- 支持画中画、分屏或解说层叠加。

### 第四阶段：存储与规模化

- 本地视频文件切换为 OSS / 对象存储。
- 素材库记录从本地路径升级为存储 key / URL。
- 支持批量任务、失败重试、队列调度和任务日志。

## 工作流概览

```text
视频线索查询
  -> 保存视频元数据
  -> yt-dlp 下载视频
  -> 素材入库
  -> Whisper 语音识别
  -> 中文翻译 / 总结字幕
  -> FFmpeg 生成处理后视频
  -> 自动剪辑增强
  -> 发布到抖音 / Bilibili
```

当前已经打通的是：

```text
视频线索查询 -> 下载 -> 中文字幕处理 -> 素材入库 -> 发布工作流
```

自动剪辑增强仍在逐步开发中。

## 本地开发

### 后端

```bash
uv sync --extra web
python sau_backend.py
```

默认后端地址：

```text
http://127.0.0.1:5409
```

### 前端

```bash
cd sau_frontend
npm install
npm run dev
```

默认前端地址：

```text
http://127.0.0.1:5173
```

如果端口被占用，Vite 会自动切换到其他端口。

### 环境变量

复制 `.env.example` 为 `.env`，并根据本地路径调整：

```env
YOUTUBE_DOWNLOAD_DIR=./videos/youtube
YOUTUBE_PROCESSED_DIR=./videos/processed
```

开发阶段视频文件不进入 Git。后续接入 OSS 后，会改为对象存储路径。

## 重要说明

- 本项目当前用于本地开发和自动化工作流验证。
- 下载、处理、发布视频前，请确保你拥有对应内容的使用权或符合平台规则。
- 当前视频文件、Cookie、数据库、`.env`、本地配置均不应提交到仓库。
- 自动剪辑能力会逐步完善，不会一次性做成复杂的全自动导演系统。

## 技术栈

- 后端：Python、Flask、SQLite
- 下载：yt-dlp
- 字幕：faster-whisper、deep-translator、ASS
- 视频处理：FFmpeg
- 前端：Vue 3、Vite、Element Plus、Pinia
- 发布：基于 social-auto-upload / uploader 模块扩展
